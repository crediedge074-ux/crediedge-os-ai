# CrediEdge Development Workflow

Every new feature follows this process.

## 1. Vision

Define the business problem.

## 2. Specification

Document the feature in `/docs`.

## 3. Implementation

Ask Codex:

> Implement the feature exactly as specified in the documentation.
> Follow the engineering principles and dashboard philosophy.
> Do not invent features that are not documented.

## 4. Review

Review every pull request before merging.

## 5. Merge

Only merge when the implementation matches the specification.
